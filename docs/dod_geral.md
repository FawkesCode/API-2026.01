# Project Definition of Done (DoD)

Para que o projeto seja considerado concluído e pronto para a entrega final, os seguintes critérios devem ser atendidos:

- [ ] **Todas as Sprints Concluídas:** Todas as Sprints planejadas foram finalizadas e suas metas alcançadas.
- [ ] **Backlog Esgotado ou Acordado:** Todos os itens do Product Backlog foram entregues ou o Product Owner concordou que o produto está completo.
- [ ] **Critérios de Aceite Finais Validados:** Os stakeholders e o PO validaram que o produto final atende a todos os critérios de aceite e requisitos de negócio.
- [ ] **Testes de Aceitação do Usuário (UAT) Aprovados:** O produto passou pelos testes de aceitação realizados por usuários finais.
- [ ] **Requisitos Não-Funcionais Atendidos:** O produto atende aos requisitos de performance, segurança e escalabilidade definidos no início do projeto.
- [ ] **Documentação Final Entregue:** Toda a documentação (técnica, de usuário, de API) está completa e foi entregue.
- [ ] **Transferência de Conhecimento (Handover):** A equipe de sustentação ou operação recebeu o treinamento e a documentação necessários para manter o produto.
- [ ] **Produto em Produção:** O produto foi implantado no ambiente de produção e está funcional.
