# User Story 12

**Como** operacional, **quero** consultar o status e o histórico dos meus pedidos de compra, **para** acompanhar o andamento das solicitações e saber se foram aprovadas ou negadas.

---

## Critérios de Aceite (CA)

### CA1: Visualização da lista dos meus pedidos de compra
**Dado** que sou um operacional autenticado no sistema  
**Quando** eu acesso a tela "Meus Pedidos"  
**Então** o sistema exibe uma lista de todos os pedidos que criei  
**E** cada pedido mostra informações resumidas (número, data de criação, status, valor total)  
**E** posso filtrar por status (pendente, aprovado, reprovado, em cotação, concluído)  
**E** posso filtrar por período (data de criação)  
**E** posso ordenar por data, número ou status  
**E** a lista exibe badges visuais indicando o status de cada pedido

### CA2: Visualização de detalhes de um pedido específico
**Dado** que sou um operacional visualizando meus pedidos  
**Quando** eu clico em um pedido para ver detalhes  
**Então** o sistema exibe todas as informações do pedido:  
**E** número do pedido  
**E** data de criação  
**E** status atual  
**E** setor e centro de custo  
**E** justificativa  
**E** lista completa de itens com quantidades e valores  
**E** valor total  
**E** dados do aprovador (se aprovado ou reprovado)  
**E** parecer do aprovador (se houver)  
**E** data de aprovação/reprovação

### CA3: Visualização de histórico de transições de status
**Dado** que sou um operacional visualizando os detalhes de um pedido  
**Quando** eu acesso a seção de histórico  
**Então** o sistema exibe uma timeline cronológica com todas as mudanças de status:  
**E** data e hora de cada transição  
**E** status anterior e novo status  
**E** usuário responsável pela mudança (quando aplicável)  
**E** comentários ou pareceres associados  
**E** o histórico está ordenado cronologicamente (mais recente primeiro)

### CA4: Notificação visual de pedidos aprovados ou reprovados
**Dado** que sou um operacional autenticado no sistema  
**E** um dos meus pedidos teve seu status alterado (aprovado ou reprovado)  
**Quando** eu acesso a tela "Meus Pedidos"  
**Então** o pedido com status alterado é destacado visualmente (ex: badge "novo")  
**E** posso identificar facilmente pedidos com atualizações recentes  
**E** o destaque permanece até que eu visualize os detalhes do pedido

### CA5: Impossibilidade de visualizar pedidos de outros usuários
**Dado** que sou um operacional autenticado no sistema  
**Quando** eu tento acessar diretamente a URL de um pedido criado por outro operacional  
**Então** o sistema exibe uma mensagem de erro "Acesso negado"  
**E** não exibe os detalhes do pedido  
**E** retorna código HTTP 403 Forbidden

### CA6: Indicador visual claro de status do pedido
**Dado** que sou um operacional visualizando meus pedidos  
**Quando** eu observo a lista de pedidos  
**Então** cada status é representado por cores e ícones distintos:  
**E** Pendente: cor amarela/laranja  
**E** Aprovado: cor verde  
**E** Reprovado: cor vermelha  
**E** Em Cotação: cor azul  
**E** Concluído: cor cinza/roxo  
**E** os ícones e cores facilitam a identificação rápida do status

### CA7: Filtro rápido por período pré-definido
**Dado** que sou um operacional visualizando meus pedidos  
**Quando** eu seleciono um filtro rápido de período (últimos 7 dias, último mês, último trimestre)  
**E** aplico o filtro  
**Então** a listagem exibe apenas pedidos criados no período selecionado  
**E** a quantidade total de pedidos no período é exibida  
**E** posso remover o filtro para ver todos os pedidos novamente
