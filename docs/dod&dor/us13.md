# User Story 13

**Como** gerente, **quero** registrar o recebimento de mercadorias vinculado ao pedido de compra, conferindo itens, quantidades e anexando a nota fiscal, **para** concluir o ciclo de aquisição e atualizar o estoque automaticamente.

---

## Critérios de Aceite (CA)

### CA1: Registro de recebimento vinculado a pedido aprovado
**Dado** que sou um gerente autenticado no sistema  
**E** existe um pedido de compra aprovado ou cotado  
**Quando** eu acesso a tela de registro de recebimento  
**E** seleciono o pedido de compra relacionado  
**E** informo a data de recebimento  
**E** confiro os itens e quantidades recebidas (podem ser parciais)  
**E** clico em "Registrar Recebimento"  
**Então** o recebimento é registrado no sistema  
**E** o status do pedido é atualizado para "Recebido" (ou "Parcialmente Recebido" se houver divergências)  
**E** o estoque é atualizado automaticamente com as quantidades recebidas  
**E** uma mensagem de sucesso é exibida

### CA2: Upload e anexação de nota fiscal
**Dado** que sou um gerente registrando um recebimento  
**Quando** eu faço upload de um arquivo (PDF, imagem) da nota fiscal  
**E** o arquivo é válido (formato aceito e tamanho dentro do limite)  
**E** clico em "Anexar Nota Fiscal"  
**Então** o arquivo é armazenado no sistema vinculado ao recebimento  
**E** o nome e data de upload do arquivo são exibidos  
**E** posso baixar o arquivo posteriormente para consulta

### CA3: Conferência de divergências de quantidade
**Dado** que sou um gerente registrando um recebimento  
**Quando** eu informo quantidades recebidas diferentes das quantidades solicitadas no pedido  
**Então** o sistema destaca visualmente as divergências (ex: cor vermelha/amarela)  
**E** me obriga a registrar uma justificativa para cada divergência  
**E** o status do pedido é marcado como "Parcialmente Recebido"  
**E** as quantidades recebidas são atualizadas no estoque, não as solicitadas

### CA4: Atualização automática de estoque após recebimento
**Dado** que um recebimento é registrado com sucesso  
**Quando** o sistema processa o recebimento  
**Então** as quantidades dos itens no estoque são incrementadas automaticamente  
**E** a atualização ocorre de forma atômica (transação)  
**E** uma entrada de movimentação de estoque é criada automaticamente (tipo: entrada, vinculada ao pedido)  
**E** alertas de estoque baixo são reavaliados

### CA5: Validação de campos obrigatórios
**Dado** que sou um gerente registrando um recebimento  
**Quando** eu tento registrar sem preencher campos obrigatórios (pedido, data de recebimento, quantidades)  
**Então** o sistema exibe mensagens de erro indicando os campos não preenchidos  
**E** o recebimento não é registrado até que todos os campos sejam preenchidos

### CA6: Impossibilidade de receber pedidos não aprovados
**Dado** que sou um gerente autenticado no sistema  
**E** existe um pedido com status "Pendente" ou "Reprovado"  
**Quando** eu tento registrar um recebimento para este pedido  
**Então** o sistema exibe uma mensagem de erro "Apenas pedidos aprovados ou cotados podem ter recebimento registrado"  
**E** o recebimento não é permitido

### CA7: Consulta de histórico de recebimentos de um pedido
**Dado** que sou um gerente autenticado no sistema  
**Quando** eu acesso os detalhes de um pedido  
**Então** o sistema exibe o histórico de recebimentos vinculados ao pedido  
**E** cada recebimento mostra: data, quantidades recebidas por item, usuário responsável, nota fiscal anexada (se houver)  
**E** posso baixar as notas fiscais anexadas  
**E** vejo se o recebimento foi total ou parcial

### CA8: Conclusão do ciclo de aquisição
**Dado** que todas as quantidades de um pedido foram recebidas  
**Quando** o último recebimento parcial é registrado  
**Então** o status do pedido é alterado para "Concluído"  
**E** o pedido não pode mais ter novos recebimentos registrados  
**E** uma notificação visual indica que o ciclo foi concluído
