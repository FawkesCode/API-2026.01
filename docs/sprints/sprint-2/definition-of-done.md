# Definition of Done (DoD) da Sprint

Para que a Sprint seja considerada concluída, todos os seguintes critérios devem ser atendidos:

- [ ] **DoD das User Stories:** Todas as User Stories no backlog da Sprint atenderam às suas respectivas Definitions of Done.
- [ ] **Código Integrado:** Todo o código novo foi integrado à branch principal (main/develop) sem conflitos.
- [ ] **Build sem Erros:** A aplicação compila e passa no processo de build sem erros.
- [ ] **Testes Automatizados OK:** Todos os testes (unitários, integração, E2E) estão passando.
- [ ] **Revisão de Código (PRs):** Todos os Pull Requests foram revisados e aprovados.
- [ ] **Demonstração (Review):** Todas as funcionalidades foram demonstradas e aprovadas pelo Product Owner na Sprint Review.
- [ ] **Documentação Atualizada:** A documentação relevante (API, arquitetura, etc.) foi atualizada.
- [ ] **Retrospectiva Realizada:** A Sprint Retrospective foi conduzida e os planos de ação foram registrados.
